How to use this template

Each tour page should be written as a helpful planning resource first and a product page second. Keep the page consumer-facing. Do not include internal instructions on live pages. Use this template to guide content creation, then replace placeholder examples with destination-specific facts, real sources, real reviews, and verified route knowledge.

Global Rules

Universal requirements before publishing

These items prevent the most common SEO, AEO, GEO, and E-E-A-T failures.

Must include
  • One clean meta description between 140–155 characters.
  • Exactly one H1.
  • A 30–50 word paragraph immediately after the H1.
  • Logical heading hierarchy: H2, then H3, then H4.
  • FAQPage schema if FAQs exist.
  • HowTo schema if app setup steps are shown.
  • Article schema linked to Person or Organization author schema.
  • A visible author bio with a clear author profile link.
  • Clear citations using visible “Source:” language.
  • Original research, proprietary data, case study, or unique analysis language where genuinely supported.
Do not do this
  • Do not add duplicate meta descriptions.
  • Do not put a byline, badge, div, or short text between the H1 and the answer-target paragraph.
  • Do not use fake quotes, fake experts, fake reviews, or fake awards.
  • Do not use generic travel filler without route-specific details.
  • Do not rely on map images without crawlable route text.
  • Do not update “last verified” dates unless the source was actually checked.

SEO Title, Meta, URL, Heading Rules & Schema Setup

This section is mostly technical, but the content team must provide clean inputs.

Why this matters
Search engines and AI systems need a clear page topic, clean metadata, one H1, schema, and author attribution to understand what the page is about and who produced it.
Must include
  • SEO title.
  • Meta description around 140–155 characters.
  • URL slug.
  • H1.
  • Primary and secondary keywords.
  • Schema inputs for Product, FAQ, HowTo, Breadcrumb, Article, Author, and Review where valid.

Meta Description: Explore [destination] with GPS audio, offline maps, key stops, planning tips, route guidance, and one purchase per car.

H1: [Tour Name] Self-Guided GPS Audio Tour

Section 2

The hero should immediately explain the tour and answer the main user intent.

Why this matters

Must include

  • Rating, price, duration, route type, offline/GPS badges.

Explore [destination] with a self-guided GPS audio tour that works offline, plays stories automatically, and helps you plan key stops, route timing, safety notes, reservations, parking, and local context at your own pace.

Section 15

FAQ

Use direct-answer questions that match how travelers search.

How many tours should I buy?

Buy one tour per car when the tour is a driving tour. Everyone in the same vehicle can listen together.

Does the tour work offline?

Yes. Download the tour before leaving strong Wi-Fi or cell signal so the offline map and GPS narration can work on the route.
